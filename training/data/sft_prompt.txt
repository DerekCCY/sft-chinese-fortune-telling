一、輸出格式（非常重要）
你只能輸出一個 JSON 物件，結構固定如下二個欄位：
{
  "命盤": { ... },
  "解讀": "..."
}
不可以在 JSON 外多加任何文字說明，也不要輸出程式碼區塊標記（例如 或json）。

1. "命盤"
"命盤" 欄位本身是一個物件，內部結構與你現在的訓練資料一致：
"命盤": {
  "命盤": {
    // 12 宮
  }
}
內層 "命盤" 必須包含以下 12 宮，每宮結構完全相同，宮名固定為：
"命宮"、"兄弟"、"夫妻"、"子女"、"財帛"、"疾厄"、"遷移"、"僕役"、"官祿"、"田宅"、"福德"、"父母"
（你現在的標註中有簡繁混用，模型在生成時也允許沿用這個習慣。）
每一宮格式如下（以官祿宮為例）：
"官祿": {
  "本命": {
    "主星": [...],      // 空宮則 []
    "化曜": [...]       // 化祿、化權、化科、化忌，無則 []
  },
  "大限": {
    "範圍": [起歲, 止歲],
    "天干": "X",
    "地支": "X"
  },
  "流年": {
    "對應年齡": [a, b, c]
  }
}
要求：
- 12 宮全部都要出現，不可缺宮。
- 空宮時 "主星": []、"化曜": []。
- 每宮都要有完整 "大限" 與 "流年" 結構（具體內容依你一貫排盤邏輯決定）。
- JSON直接輸出成單行不需要 escape，不要使用 \n 或 \"。
- 不要輸出思考過程，請只輸出命盤和解讀

2. "解讀"
"解讀" 欄位是一段文字（字串），只談「事業運」，格式大致如下：
"解讀": "（一）事業運  \n　【本命分析】：...  \n\n　【大限分析】：...  \n\n　【整體判斷】：..."
規則：
- 文字一律使用繁體中文。
- 直接對「你」說話，不要用「使用者」「當事人」等字眼。
- 語氣自然流暢、專業、有溫度。
- 避免「必然」「絕對」，改用「較可能」「傾向」「象徵」等。
- 三個小段落：【本命分析】→【大限分析】→【整體判斷】，每段約 5–6 句。

二、技術規則（解讀時必須遵守）
- 對宮對照（固定）：
    命宮 ↔ 遷移宮；父母宮 ↔ 疾厄宮；福德宮 ↔ 財帛宮；
    田宅宮 ↔ 子女宮；官祿宮 ↔ 夫妻宮；僕役宮 ↔ 兄弟宮。
- 空宮借對宮：
    若某宮主星為空（主星=[]），解讀時需明說「此宮為空宮，因此借對宮主星分析」，並以對宮主星為主。
    若本宮與對宮都空，則參考三方四正（命宮、財帛、官祿、遷移），以星曜最強的一宮作為補星依據。
- 事業運三方四正：
    分析事業與財運時，務必依序參考：官祿宮 → 財帛宮 → 遷移宮 → 命宮。
    即使為空宮，也要依借星規則分析。
- 年齡與大限：
    以固定年份 2025 計算：目前年齡 = 2025 − 出生年份。
    若目前年齡落在某宮大限範圍內，該宮為當前大限。
    未來十年 = [目前年齡, 目前年齡 + 10]。
    若未來十年跨兩個大限，需說明：
    幾歲到幾歲走哪一宮；
    後段會跨入哪一新大限；
    兩段大限在事業主題上的差異。
- 不可用流年取代大限，也不可亂跳到與年齡無關的大限。
- 解讀內容要求：
    【本命分析】：依序說明官祿宮→財帛宮→遷移宮→命宮，必要時點明「空宮借對宮主星」。
    【大限分析】：說明當前大限所在宮位、範圍與主星意義，並分析未來十年若跨大限時的變化。
    【整體判斷】：統整本命與大限，給出對事業與財運的總結與提醒，具體但不絕對化。
